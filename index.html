How To Use This If You Have Never Programmed

Do not skim. Programming is learned by reading a little, typing code, running it, breaking it, and fixing it.

Week A course unit. Week 1 is where you start writing C. Do one week at a time.
Lecture notes The explanation part. Read slowly, then type the code example yourself.
Section walkthrough A guided practice explanation. This is where ideas become small working programs.
Shorts / syntax drills Small memorization targets. These help you remember punctuation and common patterns.
Practice assignment CS50 calls this a problem set, or pset. It means programming homework where you build real programs.
Compile Turn your C source code into a runnable program. In CS50, you usually do this with make program.
void Means “nothing” in a specific place. In main(void), it means no inputs. Before a helper function, it means no returned value.
string CS50's beginner type for text. It comes from cs50.h. Text directly inside quotes is a string literal.
Placeholder A marker like %s or %i inside a printf format string. Later arguments fill those markers in order.

Mental Model

How to think while writing C

C is strict because the compiler and runtime do almost exactly what you asked, not what you intended. The learning progression is: control syntax, then arrays and strings, then algorithmic reasoning, then memory ownership, then pointer-based structures.

Every name needs a type. Before a variable can hold data, C needs to know what kind of data it stores.
Every external function needs a declaration. Headers such as stdio.h and cs50.h tell the compiler which functions exist.
Every branch should be testable. For each if, choose input that proves it runs and input that proves it does not run.
Every loop needs progress. Ask: what changes each iteration, and what eventually makes the condition false?
Every pointer needs ownership clarity. Ask: who allocated this memory, who is allowed to use it, and who frees it?
Every data structure is a tradeoff. Arrays, lists, tables, and tries buy different lookup, insertion, deletion, and memory costs.
